# Failure Analysis

본 문서는 최종 평가에서 확인된 주요 실패 유형과 개선 방향을 정리한 문서입니다.

## 1. 사용자 표현과 법령 표현 차이

### 사례

```text
운전하면서 휴대폰 보면 처벌받아?
```

### 문제

사용자 표현인 `휴대폰`과 법령 표현인 `이동전화`, `영상표시장치`가 충분히 매핑되지 않아 직접 근거 조문 검색에 실패했습니다.

### 개선 방향

- 법령 도메인 동의어 사전 구축
- 사용자 표현과 법령 표현 매핑
- 핵심 키워드 미포함 문서에 대한 rule-based filtering 적용

---

## 2. 질문 유형 구분 부족

### 사례

```text
스쿨존에서 속도위반하면 어떻게 돼?
```

### 문제

질문은 속도위반의 결과를 묻고 있지만, 검색 결과는 어린이 보호구역 제한속도 조문에 치우쳤습니다.

### 개선 방향

- 규정 확인형, 위반 처분형, 신고 절차형 질문 분류
- `위반`, `처벌`, `과태료`, `범칙금`, `벌점` 키워드 기반 검색 가중치 조정

---

## 3. 조문 번호 오인용

### 문제

검색 결과는 적절했지만 LLM이 검색 context와 다른 조문 번호를 생성하는 경우가 있었습니다.

### 개선 방향

- LLM이 조문 번호를 직접 생성하지 않도록 제한
- 검색 문서의 metadata에서 법령명과 조문 번호를 자동 삽입
- 생성 후 조문 번호 검증 로직 추가

---

## 4. 무관 문서 혼입

### 문제

Retriever 단계에서 무관 문서가 후보에 포함되면 Reranker만으로 완전히 보정하기 어려웠습니다.

### 개선 방향

- Reranker 이후 rule-based filtering
- 질문 핵심 키워드와 문서 metadata 기반 필터링
- 법령명, 조문명, article metadata 활용 강화
